Coding Color Schemes - SQL Management Studio

Mon, Jan 1, 0001 • ∞ http://joshua.poehls.me/1/coding-color-schemes---sql-management-studio

Spending all day looking at the default color schemes in Visual Studio started making me go buggy-eyed several months ago. Mostly it was the white background - just way to bright for a programmer who does his best work in the dark.

So… taking cues from others who have ‘gone black’ with their color scheme, I set out to customize my coding environment. The result? See for yourself… here is the color scheme I did for SQL Management Studio. I’ll post my VS.NET settings later but they are pretty similar.

  • Changed font to Monaco in Environment|Fonts and Colors under “Text Editor”
  • Plain Text - White (White)
  • Comment - (R,G,B) 74,255,74 (Greenish)
  • Keyword - (R,G,B) 177,100,255 (Purplish)
  • Memory Data - White
  • Number - (R,G,B) 255,108,108 (Redish)
  • SQL Operator - (R,G,B) 43,145,175 (Bluish)
  • SQL Stored Procedure - (R,G,B) 255,108,108 (Redish)
  • SQL String - (R,G,B) 255,108,108 (Redish)
  • SQL System Function - (R,G,B) 255,255,117 (Yellowish)
  • SQL System Table - (R,G,B) 74,255,74 (Greenish)
